## 测试方法

### 单元测试策略

所有规则测试基于 `@parser.parse_string()` 构造 AST 进行验证，无需依赖真实文件系统：

```moonbit
test "CWE-116: replace with HTML metachar should trigger" {
  let source = "fn foo(s : String) -> String { s.replace(old=\"<\", new=\"&lt;\") }"
  let (impls, _) = @parser.parse_string(source, name="test.mbt")
  let findings : Array[Finding] = []
  run_cwe116(impls, "test.mbt", source, findings, Error)
  assert_true(findings.length() > 0)
}
```

这种方式确保：
- 测试隔离性：不依赖外部文件或环境
- 可重复性：同一输入始终产生同一结果
- 跨平台：wasm、js、native 编译目标下行为一致

### 集成测试

通过 CLI 对真实项目进行端到端扫描验证：

```bash
# 扫描真实项目
moon run src/main -- /path/to/mocket

# LLM 分析流程
moon run src/main -- llm-analyze --format script /path/to/mocket
python3 llm_analyze.py

# PoC 生成
moon run src/main -- generate-poc /path/to/mocket

# 修复建议
moon run src/main -- remediate /path/to/mocket

# 统计报告
moon run src/main -- summary /path/to/mocket
```

### CI 持续集成

GitHub Actions 配置（`.github/workflows/check.yml`）在每次推送时自动运行：

```yaml
strategy:
  matrix:
    os: [ubuntu-latest, macos-latest, windows-latest]

steps:
  - moon check          # 编译检查
  - moon test           # 运行全部测试
  - moon fmt && git diff --exit-code   # 格式一致性
  - moon info && git diff --exit-code  # 接口文件一致性
```
