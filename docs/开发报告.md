# 开发报告

## 项目概述

moon-audit 是面向 MoonBit 生态的 AST 级静态安全扫描器，检测通用安全问题和 Web 框架漏洞模式。基于对 MoonBit 生态 7 个开源项目的深度安全审计成果，提炼出 14 条 CWE 检测规则。

- **GitHub**：https://github.com/I3eg1nner/moon-audit
- **Gitlink**：https://gitlink.org.cn/i3eg1nner/moon-audit
- **mooncakes.io**：`minie135/moon-audit` v0.1.0
- **许可证**：MulanPSL-2.0

## 技术路线

| 阶段 | 内容 | 产出 |
|---|---|---|
| 生态调研 | MoonBit 语言特性、编译目标、标准库和第三方生态安全调研 | 识别 Web 应用场景关键攻击面 |
| 漏洞审计 | 7 个开源项目深度安全审计 | 49 个运行时可复现漏洞，4 个安全修复 PR |
| 扫描器开发 | 基于审计成果提炼 AST 级检测规则 | moon-audit 静态安全扫描器 |

## 核心技术

### AST 规则匹配

基于 `moonbitlang/parser` 的 `IterVisitor` trait，每条规则实现为独立 Visitor。覆写 `visit_Expr_DotApply`、`visit_Expr_Apply`、`visit_Expr_Guard` 等方法匹配漏洞模式。

### Import 门控

解析 `moon.pkg`/`moon.mod` 构建 `ImportContext`，仅在项目引入相关框架时激活对应 Web 规则。

### 上下文感知过滤

| 规则 | 过滤策略 | 降噪效果 |
|---|---|---|
| CWE-676 | 仅报告危险类型转换，跳过性能操作和 guard body | async 100→3 |
| CWE-248 | 跳过裸 panic、guard-else、平台桩、unreachable、契约断言 | core 104→28 |
| CWE-704 | 跳过 FFI 绑定文件 | mocket 22→0 |
| 标准库白名单 | moonbitlang/core 等自动跳过通用规则 | core 63→0 |

### LLM 辅助分析

MoonBit native 目标不能直接发 HTTP，采用 Python 脚本桥接：MoonBit 端解析 `.env` 配置、生成结构化提示词，输出自包含 Python 脚本调用 LLM API 逐条验证 Finding 真伪。兼容 Anthropic Claude 和 OpenAI 兼容 API。

## MoonBit 语言适配

开发中遇到的主要语言特性问题：

| 问题 | 解决方案 |
|---|---|
| `String[i]` 返回 `UInt16` 非 `Char` | 实现 `char16()` 辅助函数 |
| `#\|` 多行字符串不处理插值 | 测试中用 `#\|` 避免 `\{...}` 被编译器处理 |
| `String.substring()` 已弃用 | 使用 `s[start:end].to_owned()` 切片语法 |
| `method` 是保留关键字 | 重命名为 `mname` |

## 已知限制

- **Parser 版本**：依赖 `moonbitlang/parser` 0.3.9，不支持部分较新语法（如 `with` 默认值绑定），使用新语法的源文件会被跳过
- **CWE-704 误报率高**：`.cast()` 在 FFI 重度项目中是唯一类型转换手段，~98% 误报率，已默认关闭
